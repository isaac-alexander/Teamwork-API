CONTROLLER - Contains REST API endpoints.

Controllers receive HTTP requests from clients and return responses.

SERVICE - Contains the business logic of the application.

Services process requests before interacting with the database.

REPOSITORY - Contains database access code.

Repositories communicate with MySQL using Spring Data JPA.

ENTITY - Contains database models.

Each entity becomes a table inside the database.

DTO - Contains Data Transfer Objects.

DTOs are used to send and receive data without exposing database entities.

MAPPER - Converts Entity objects into DTOs and DTOs into Entity objects.

SECURITY - Contains JWT authentication classes and filters.

CONFIG - Contains application configuration classes such as SecurityConfig and SwaggerConfig.

Application Flow

Client - Controller - Service - Repository - Database - Repository - Service - Mapper - Controller - Client

Annotations

@SpringBootApplication

Marks the main class of the application.

It combines three annotations:

@Configuration

@EnableAutoConfiguration

@ComponentScan

@RestController

Marks a class as a REST controller.

Automatically returns JSON responses.

@RequestMapping

Defines the base URL for a controller.

Example

/api/articles

@GetMapping

Handles HTTP GET requests.

@PostMapping

Handles HTTP POST requests.

@PutMapping

Handles HTTP PUT requests.

@DeleteMapping

Handles HTTP DELETE requests.

@PathVariable

Reads values directly from the URL.

Example

/api/articles/1

The value 1 becomes the article ID.

@RequestParam

Reads query parameters.

Example

?page=0&size=5

@RequestBody

Converts JSON into a Java object.

Example

{
    "title":"Spring Boot",
    "content":"Learning Spring Boot"
}

@ResponseStatus

Returns a specific HTTP status code.

Example

201 Created

@RequiredArgsConstructor

Generated by Lombok.

Automatically creates a constructor for all final fields.

This allows constructor dependency injection without writing constructors manually.

Database

The application uses MySQL.

Spring Data JPA automatically creates tables from Entity classes.

Configuration is stored inside application.yml.
